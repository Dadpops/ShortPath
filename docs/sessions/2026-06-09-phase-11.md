# 2026-06-09 ‚Äî Phase 11: nested sub-folders, delete vertical, check for updates

## What was done

### Nested sub-folders

`SubFolder` gained an optional `subFolders?: SubFolder[]` field for arbitrary nesting depth.

Store layer (`store/index.ts`) added four private tree helpers:
- `addToTree` ‚Äî inserts a new node under a given parent ID (or at root if none)
- `renameInTree` ‚Äî finds and renames a node at any depth
- `removeFromTree` ‚Äî prunes a node and all its descendants
- `collectSubtreeIds` ‚Äî collects all IDs under a root node (used by `removeSubFolder` to clear entry assignments)

`addSubFolder` now accepts an optional `parentSubFolderId`. `renameSubFolder` and `removeSubFolder` use the new helpers instead of flat-array operations.

CSV (`csv.ts`) gained a `subfolder` column in both import and export. On export, `exportCsv` receives the `verticals` array so it can resolve `subFolderId` to a human-readable label. On import, labels are matched case-insensitively against existing sub-folders; unknown labels auto-create a new top-level sub-folder in the appropriate vertical.

`FolderIcon.tsx` is a new SVG component that renders a closed or open folder in `--color-accent` stroke, used in `VerticalGroup` sub-folder headers.

`VerticalGroup.tsx` renders sub-folders recursively. Each node has a chevron, a `FolderIcon`, a label, and a total count (including all descendants). Nesting is expressed with a fixed 14px left margin per depth level. All nodes start expanded (using `getAllIds` to pre-populate the `expandedSubs` set).

`EntryForm.tsx` flattens the sub-folder tree to a depth-indented `<select>` using `‚Ü≥` prefixes for nested levels.

`SettingsScreen.tsx` received a new sub-folder tree management UI in the Organization section. Each top-level sub-folder node shows a collapse/expand toggle (when it has children), an edit button, a remove button, and an "add sub-folder" button that places the new node under the current one. Depth is expressed with `paddingLeft: depth * 16px`.

### Delete vertical

`deleteVertical` was added to `store/index.ts`. It removes the vertical from the `verticals` array and strips all entries whose `vertical` field matches the ID.

The IPC handler `delete-vertical` calls `deleteVertical`, saves the store, removes the vertical from `settings.verticalOrder` if present, saves settings, and pushes a store update.

`SettingsScreen.tsx` shows an inline confirmation row when the user clicks "Delete" next to a vertical: "Delete [name] and all its entries?" with a red confirm button and a cancel link.

### Check for updates

`checkForUpdates()` in `main/index.ts` hits `https://api.github.com/repos/Dadpops/ShortPath/releases/latest` using Node's built-in `https` module (no extra dependency). It compares `tag_name` (stripped of leading `v`) to `app.getVersion()` using a semver-style numeric comparison. Returns `{ version, url }` if a newer version exists, or `null` if already current or on error. The request has a 10-second timeout.

On app ready (after a 5-second delay to avoid slowing startup), the result is pushed to the renderer via `win.webContents.send("update-available", update)`. The IPC channel `check-for-updates` also lets the renderer trigger a manual check.

`App.tsx` subscribes to `onUpdateAvailable` and shows a dismissable banner below the header when an update is found. The banner shows the version number and a "Download" link that opens the release page via `shell.openExternal`.

`SettingsScreen.tsx` (About section) has a "Check for updates" button that calls `checkForUpdates` and shows either "Up to date" (auto-dismissed after 3s) or a "v{version} available ‚Äî Download" link inline.

### Other improvements

**Favorites on Support Tools**: `SupportToolsGroup` now accepts `favorites` and `onToggleFavorite` props; each tool entry gets a star toggle for parity with other result types.

**Search bar clear icon**: The text `√ó` button was replaced with an SVG X icon that occupies the same position as the search icon. The button appears when the input is focused or non-empty, and blurs the input when clicked (rather than re-focusing it).

**Vertical tab overflow**: When more than 5 verticals exist, the tab row switches to a `<select>` element to avoid wrapping.

## Decisions

- Tree helpers are private functions in `store/index.ts` (not exported). The public API is the same four store functions as before; nesting is an implementation detail.
- `removeSubFolder` now collects the entire subtree of IDs before deletion so entries in nested children are also unassigned (not just direct children).
- `deleteVertical` does not prompt for synced entries ‚Äî if a vertical is deleted, all its entries (local and synced) are removed. The UI confirmation message says "and all its entries."
- `checkForUpdates` uses the built-in `https` module rather than `electron-updater` or `node-fetch` to avoid adding a runtime dependency for a lightweight check.
- The auto-check fires 5 seconds after `app.whenReady` rather than immediately to avoid racing the window paint.
- The vertical tab/select threshold is 5 (switches when `orderedTabVerticals.length > 5`), chosen so that the typical user with 4‚Äì5 verticals never sees a select.
