## Decisions made

- **Clipboard banner replaced by header icon, not removed.** The banner added noise below the search area. Moving it to an icon in the header keeps the signal without cluttering the search UX. The icon pulses (CSS animation) so it's noticeable without being intrusive. Dismissal behavior is unchanged — clicking the icon opens the add form and sets dismissed=true; re-reading on window focus resets it.
- **Import/Split stay as AppModes; Settings navigates to them.** The Import and Split screens have multi-step UIs that don't fit inside the Settings panel. Rather than rebuild them inline, Settings has buttons that navigate to those modes via `onNavigate`. This avoids duplicating logic.
- **SupportToolsGroup used for the `support-tools` vertical only.** The distinct grid layout is for the dedicated Support Tools category. Tool-type entries in other verticals render normally. The check is `group.verticalId === "support-tools"`, not `entry.type === "tool"`.
- **Reorder arrows hidden during search.** When searching, results are in relevance order, not store order. Showing reorder arrows during search would be confusing — the position changes would apply to the store order (not the visible search order). Arrows only appear in browse mode.
- **Help panel is a mode swap, not a slide-over.** Given the app is a small popup, the visual distinction between a slide-over and a mode swap is minor. The mode swap is simpler to implement and doesn't require animation or z-layering. The Back button provides clear navigation.
