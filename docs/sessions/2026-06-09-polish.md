# Session: Polish and Bug Fixes — 2026-06-09

## Summary

Full QA runthrough of the app after Phase 14. Found and fixed a fatal crash that prevented the app from launching, four TypeScript build errors, a missing Sync UI, and two visual issues.

## What changed

### Fatal crash: ERR_REQUIRE_ESM
`jsdom` v24+ and its dependency `html-encoding-sniffer` are ESM-only packages. They cannot be `require()`'d in Electron's CommonJS main process, so the app crashed on launch before creating any window.

Fix: removed `jsdom`, `@mozilla/readability`, and `marked` from the main process entirely. The `fetch-url-content` IPC handler now returns raw HTML only. All parsing (DOMParser + Readability + splitByHeadings) was moved to the renderer, which runs in a browser context and has native DOM support.

Files changed:
- `src/main/index.ts`: removed ESM imports, removed `splitByHeadings` and `UrlSection` interface, simplified handler
- `src/renderer/components/EntryForm.tsx`: added `Readability` import, rewrote `handleFetchUrl` and added `splitByHeadings`
- `src/renderer/global.d.ts`: updated `fetchUrlContent` return type to `{ html: string; finalUrl: string } | { error: string }`

### TypeScript build errors (4 fixed)
1. `marked` unused import in main — removed
2. Regex narrowing: `((h2 ?? h3)![1])` — `h2` can be `false`, so TypeScript can't index it; changed to `(h2 ? h2[1] : h3![1]).trim()`
3. `article.content` nullable — guarded with `??`
4. `article.title` nullable — guarded with `??`

### Sync management missing from UI
The IPC layer for sync (configure-sync, refresh-synced, disconnect-sync, get-sync-status) was fully wired in main and preload, but the Settings screen had no UI to access it after the onboarding wizard was bypassed.

Fix: added a Sync section to `SettingsScreen.tsx` between Organization and Data. Shows path, synced count, last refreshed date, and Refresh/Change/Disconnect buttons when connected; "Configure sync file" button when not connected.

Files changed:
- `src/renderer/components/SettingsScreen.tsx`: added sync state, handlers, and JSX section
- `src/renderer/styles/global.css`: added sync section styles

### URL import button styling
The "Import from URL" toggle button rendered as an unstyled white platform button in dark mode. Fixed by styling it as an accent-colored underlined text link.

Files changed:
- `src/renderer/styles/global.css`: added `.url-import-toggle`, `.btn-link`, `--color-error` token

## Decisions

- **All parsing in renderer**: DOMParser is native in browser contexts. No JSDOM needed. Main process handles only network I/O.
- **Sync UI in Settings, not onboarding**: SetupScreen is dead code (never rendered). Sync management belongs in Settings where users naturally look for it.
