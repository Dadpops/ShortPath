# 2026-06-08 — Phase 10: customization and polish

## What was done

### Section 2 — Accent color and window personality

**2.1 Accent color picker**

Added 6 preset swatches (Ocean, Violet, Rose, Amber, Teal, Slate) to Settings > Appearance. Each swatch is a 22px circle; the active swatch shows a 2px ring in its own color via CSS outline. Clicking sets `--color-accent` and recomputes `--color-accent-dim` as a low-opacity rgba of the chosen color. Color persists to `settings.accentColor` and is applied on launch via `applyAccent()` in App.tsx.

The default `--color-accent` in global.css was updated from `#3b82f6` to `#2563eb` to match the Ocean preset spec value.

**2.2 Window opacity**

Added opacity slider (70–100, default 100) in Settings > Appearance. Label shows current % value. `set-opacity` IPC calls `win.setOpacity(value/100)`. Persists to `settings.opacity`. Restored on launch via the `opacity` option in `new BrowserWindow(...)`.

Note: the BrowserWindow `opacity` constructor option is set at window creation. Dynamic changes use `win.setOpacity()` via IPC.

**2.3 Window size presets**

Added Small/Medium/Large buttons in Settings > Appearance (380×520 / 480×640 / 580×760). `set-window-size` IPC calls `win.setSize()` and persists to `settings.windowSize`. On launch, if `windowSize` is set the preset dims are used instead of `windowBounds` dims (position still comes from `windowBounds`). Updated `WINDOW_WIDTH`/`WINDOW_HEIGHT` constants from 512×632 to 480×640 to match the Medium preset.

### Section 3 — Layout and density

**3.1 Compact density**

Added Compact/Comfortable toggle in Settings > Appearance. On change, sets `data-density="compact"` on `<body>` and persists to `settings.density`. Restored on launch. CSS block targets `body[data-density="compact"]` to reduce row padding, group header padding, body text font size, and tags font size — no duplication of component styles.

**3.2 Custom vertical tab ordering**

Added "Tab order" draggable list in Settings > Organization. Uses HTML5 Drag and Drop API — no library. Each row shows drag handle `⠿`, vertical name, and entry count. `onDrop` reorders the array and calls `set-vertical-order` IPC. Persists to `settings.verticalOrder`. App.tsx reads `verticalOrder` from `load-entries` and applies it to both the vertical tab bar and the `rawGroups` computation.

### Section 4 — UX upgrades

**4.1 Pinned entries**

Added `pinned: string[]` to `StoreData`. `togglePin` and `incrementCopyCount` functions added to `store/index.ts`. `toggle-pin` IPC enforces max 8 — returns `{ ok: false, limitReached: true }` if at limit. Pin button (📍/📌) added to `ResultItem` and `MacroOverlay`. Pinned section renders above all groups when search is empty. Pin limit message shows inline for 3 seconds. During active search, pins render normally within results.

**4.2 Keyboard navigation**

- Arrow Down/Up already existed; wrapping preserved.
- Enter on focused row opens overlay (already existed).
- Enter while overlay is open now copies body — added to `MacroOverlay.tsx` keydown handler at capture phase.
- Escape cascade already existed (overlay → clear search → hide).
- Tab cycles vertical filter tabs — handled in window-level keydown when `mode === "browse"` and focus is not in an input.
- Focused row now uses left-border accent + `var(--color-accent-dim)` background instead of outline.
- Search input auto-focused on window show (existing `focusTrigger` mechanism).
- Arrow Down from search moves to first result (existing `onNavigateDown` prop).

**4.3 Copy then auto-hide**

Added "Hide window after copying" On/Off toggle in Settings > Behavior. `set-auto-hide-on-copy` IPC persists. In `handleCopy` in App.tsx, if `autoHideOnCopy` is true, `window.shortpath.hideWindow()` is called after 300ms. Setting propagated via `onAutoHideOnCopyChange` prop from App to SettingsScreen.

**4.4 Global hotkey**

Existing hotkey UI moved from its own section into Behavior group. No behavior changes.

### Section 5 — Usage tracking and sort

**5.1 Entry usage counter**

`copyCount?: number` added to `Entry` type. `incrementCopyCount` store function increments only `source === "local"` entries. `increment-copy-count` IPC calls it and pushes store update. `handleCopy` in App.tsx checks `entry.source === "local"` before calling. Count badge `"12×"` renders in `ResultItem` next to title, using `.copy-count-badge` (muted, monospace, only when > 0).

**5.2 Sort control**

`<select>` element added in `.sort-bar` between search and results. Options: Relevance / Most used / Recently added / A to Z. Session-only state (`useState` in App.tsx, not persisted). When no query active and sort is Relevance, `rawGroups` sorts by `most-used` (copy count). The dropdown label shows "Most used" when Relevance is selected and no query is active, giving the user honest feedback about the actual sort.

**5.3 Recent copies list**

`sessionCopies: string[]` state (max 5, in-memory only). Updated in `handleCopy`. "Recent" section renders below Pinned when search is empty and `sessionCopies.length > 0`. Uses same `ResultItem` component as normal results.

### Section 6 — Settings screen reorganization

SettingsScreen.tsx reorganized from Appearance / Data / Verticals / Global hotkey / Window into:

- **Appearance** — text size, theme, accent color, opacity, window size, density
- **Behavior** — copy then auto-hide, summon hotkey, window reset
- **Organization** — tab order (new), verticals management
- **Data** — unchanged

Each group uses the existing collapsible `.settings-section` pattern. Intra-group structure uses `.settings-group-label` (small all-caps muted label + divider) to separate sub-areas within Organization. No cards, no boxes — consistent with existing minimal style.

## Decisions

- `--color-accent-dim` is computed dynamically as `rgba(r,g,b,0.15)` in dark mode and `rgba(r,g,b,0.12)` in light mode. This makes all six accent presets work without per-preset dim values.
- `copyCount` increment is skipped for synced entries at the IPC handler level (checks `entry.source !== "local"` before modifying).
- Tab key cycling only fires in `mode === "browse"` and when focus is not in an input — Settings and form flows are unaffected.
- The sort option labeled "Relevance" actually applies "Most used" order when no query is active. The dropdown label updates to "Most used" in that case to reflect the real behavior.
- Window size presets store the choice in `settings.windowSize`. On launch, if `windowSize` is set, preset dims are used for width/height; otherwise `windowBounds` dims are used. Position always comes from `windowBounds`.
- The existing "Global hotkey" and "Window" Settings sections were folded into the new "Behavior" group rather than kept as separate top-level sections.
