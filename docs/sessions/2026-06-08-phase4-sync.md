# Session — 2026-06-08 (Phase 4 Shared-file Sync)

## Goal

Implement Phase 4: shared-file sync. Teams drop a CSV in Dropbox/Drive/OneDrive; every ShortPath instance watches it, reloads on change, and merges without ever touching local entries.

## What changed

- **src/shared/types.ts** — added IPC constants: `CONFIGURE_SYNC`, `REFRESH_SYNCED`, `CLEAR_SYNCED`, `GET_SYNC_STATUS`, `SYNC_REFRESHED`.

- **src/main/settings.ts** — added `syncPath?: string` to `AppSettings`. Persisted in `settings.json` so the configured path survives restarts.

- **src/store/csv.ts** — added `parseSyncedCsv(csvString)`. Parses a CSV and returns `Entry[]` with `source: "synced"`. Uses a deterministic ID (`synced:${vertical}:${title}`) when the row has no `id` column, so recents survive a re-sync. If a row has an `id` field (round-trip from export), that ID is used instead. Same validation as the user import path (required columns, valid type values).

- **src/store/index.ts** — added `replaceSyncedEntries(store, newSyncedEntries)`. Filters out all current `source === "synced"` entries, appends new ones, adds any missing verticals, and prunes recents that no longer exist. Local entries are never read, written, or deleted by this function — the guarantee is enforced here, not in callers.

- **src/main/index.ts** — added sync state variables: `syncWatcher`, `syncDebounceTimer`, `lastRefreshed`. Three helpers: `loadSyncedFile(path)` (parse + replace + push), `startSyncWatcher(path)` (fs.watch with 500ms debounce), `stopSyncWatcher()` (closes watcher and timer). Four IPC handlers: `configure-sync` (file picker, save path, load, start watcher), `refresh-synced` (re-read configured path), `clear-synced` (replace synced entries with empty array), `get-sync-status` (returns path, count, lastRefreshed timestamp). On startup: if `settings.syncPath` exists and the file is present, loads it and starts the watcher. Stops watcher on `will-quit`.

- **src/preload/index.ts** — exposed: `configureSync`, `refreshSynced`, `clearSynced`, `getSyncStatus`, `onSyncRefreshed`.

- **src/renderer/global.d.ts** — added types for all new sync IPC methods.

- **src/renderer/components/SettingsScreen.tsx** — replaced the stub sync section with a real UI. On mount, calls `getSyncStatus()` and subscribes to `onSyncRefreshed` to keep the display current when the file watcher fires. Unconfigured state: description + "Configure sync file" button. Configured state: sync file path, entry count, last-refreshed time (relative: "2m ago"), Refresh now, Change file, Clear synced buttons. Clear synced is a two-click confirm ("Clear synced" → "Confirm clear") with a warning note showing the entry count. Refresh button shows "Refreshing…" during the call and "Refreshed ✓" for 2s on success.

- **src/renderer/components/ResultItem.tsx** — added a small `"synced"` badge (`<span className="result-source-badge">`) on entries where `entry.source === "synced"`. No badge on local entries (noise reduction).

- **src/renderer/styles/global.css** — added: `.result-source-badge` (monospace, accent-colored, same style as macro vertical badge), `.settings-row-stack` (column flex for path display), `.settings-sync-path` (monospace, muted, word-break), `.settings-sync-actions` (flex-wrap row for action buttons), `.settings-danger-btn` (red tint for destructive action).

## Decisions made

- **`fs.watch` over `chokidar`.** `chokidar` is more reliable across platforms but adds a dependency. `fs.watch` is built into Node.js and is sufficient for watching a single file. If users report missed updates (common on network drives), `chokidar` can replace it without changing the calling code.
- **Deterministic IDs for rows without an id column.** When a team admin creates the shared CSV manually without exporting from ShortPath, there are no `id` columns. Using `synced:${vertical}:${title}` as a stable ID means re-syncing the same file won't invalidate any recents entries. Round-tripped files (exported from ShortPath) keep their original UUIDs.
- **`replaceSyncedEntries` in the store module, not in callers.** The CLAUDE.md guarantee ("sync never modifies local entries") is enforced at the function level. All callers pass arbitrary `Entry[]` arrays and don't need to pre-filter — the function handles it.
- **Two-click confirm for Clear synced.** Removing all synced entries is a large destructive action. Showing the entry count in the warning ("removes all 47 synced entries") makes it harder to click through accidentally.
- **`sync-refreshed` push event.** When the file watcher fires and reloads, `store-updated` already pushes the new entries to the renderer. A separate `sync-refreshed` event is needed for SettingsScreen to re-fetch the status (count, lastRefreshed timestamp) without polling.
- **"synced" badge only on synced entries (not "mine" on local).** Showing "mine" on every local entry would be visual noise with no information value. The badge exists to surface entries that came from a team file; that's only relevant when synced entries are present.
