# 2026-06-09 — v0.2.0: chokidar, electron-updater, release, screenshots

## What was done

### chokidar sync watcher

Replaced `fs.watch` with chokidar 3 (CJS-compatible; chokidar 4+ is ESM-only and not compatible with Electron's CommonJS main process). Key options used:

- `persistent: false` — watcher does not keep the process alive on its own
- `awaitWriteFinish: { stabilityThreshold: 300, pollInterval: 100 }` — waits for the file to stop changing before triggering the reload; eliminates partial-read race on slow network writes
- `ignoreInitial: true` — does not fire on first watch

`stopSyncWatcher` now calls `void syncWatcher.close()` since chokidar's `close()` returns a Promise (unlike `fs.FSWatcher.close()` which is synchronous).

The "Network drive file watching" known limitation from v0.1.0 is resolved.

### electron-updater scaffold

Replaced the manual `https`-based GitHub API check with `electron-updater`. Configuration:

- `autoUpdater.autoDownload = false` — user must explicitly click Download; no silent background downloads
- `autoUpdater.autoInstallOnAppQuit = true` — if a download completes, it installs on next quit
- Auto-check fires 5 seconds after launch, packaged builds only (`app.isPackaged` guard prevents errors in dev)
- `update-available` event pushes `{ version, url }` to renderer via `win.webContents.send`
- `update-downloaded` event pushes to renderer so banner changes from "Download" to "Restart & Install"

New IPC handlers: `check-for-updates` (manual trigger from Settings), `download-update`, `install-update`.

`electron-builder.yml` gains a `publish` block:
```yaml
publish:
  provider: github
  owner: Dadpops
  repo: ShortPath
```
This tells electron-updater where to look for `latest.yml` on update checks.

App.tsx banner flow: "Version X available → Download → [downloading] → Restart & Install".

### v0.2.0 release

Bumped `package.json` version from 0.1.0 to 0.2.0. Rebuilt Windows installer (`npm run dist:win` → `ShortPath Setup 0.2.0.exe`). The `release/latest.yml` artifact is included in the GitHub release so electron-updater in existing installs can detect this update.

## Decisions

- chokidar 3 instead of 4 because chokidar 4 is ESM-only; Electron main process targets CJS via vite-plugin-electron.
- `electron-updater` replaces the manual GitHub API check entirely rather than running both in parallel. The manual check was a temporary scaffold; electron-updater covers both the in-app download flow and the update notification.
- `autoDownload: false` because silently downloading a 79 MB installer without user confirmation is poor UX for a tray app.
- `latest.yml` included as a release asset alongside the installer so electron-updater in v0.2.0 installs can discover future versions.
- Version bumped to 0.2.0 (not 0.1.1) because Phase 10 + Phase 11 represent substantial new features, not a patch.
