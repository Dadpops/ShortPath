# 2026-06-09 — Phase 12: onboarding, import UX, notes save, collapse-all, export-selected

## What shipped

### SetupScreen onboarding wizard
- 4 steps: source mode → (sync folder name) → optional CSV import → "You're ready"
- Local flow skips step 2; sync flow includes it
- Step 3 reuses the drag-drop import zone (same IPC as ImportScreen)
- Step 4 shows quick-start tips

### Drag-drop CSV import + column mapping
- ImportScreen accepts a dragged .csv file via `File.path` (Electron native path)
- `stageCsvFile` IPC stages the file without opening a native dialog
- If required headers (title, vertical, type) are missing after normalization, returns `needsMapping: true` and the raw column names
- Manual column mapping UI: user assigns each required + optional field to a CSV column via dropdowns
- `previewCsvWithMapping` applies the mapping and returns a standard preview
- `commitCsvImport` uses `importCsvWithMapping` when a mapping is stored

### Notes Save button
- Explicit Save button added to the notes editor footer
- Debounced auto-save (1500ms) still runs; Save button triggers immediate save
- Status badge shows "Saving…" / "Saved" feedback; "Saved" clears after 2s
- Confirm-delete button text changed from "Delete" to "Confirm delete" for clarity

### Text size fix
- All 119 hardcoded `font-size: Xpx` values replaced with CSS variable expressions (`var(--font-size-xs)` etc.)
- Seven helper vars added to `:root`: `--font-size-2xs` through `--font-size-2xl`, all derived from `--font-size-base`
- Live px indicator added next to the slider in Settings Appearance group

### Sub-folder CSV import fix
- Documented `subfolder` column in ImportScreen format table and help topics
- Fixed expand-after-import bug: `onStoreUpdated` handler now merges new vertical IDs into `expandedGroups`

### Collapse-all toggle
- `⊟ Collapse` / `⊞ Expand` button in the results sort bar
- Toggles all vertical groups at once; tracks state via `expandedGroups`
- Only shown when there are results

### Export Selected screen
- Replaces "Export mine" in Settings with "Export selected"
- Full-screen checkbox tree: vertical → sub-folder → individual entry
- Indeterminate checkbox state for partial-selection (set via `ref` callback)
- "All" / "None" header buttons; footer shows selected count + Cancel + Export buttons
- `export-selected` IPC handler filters store entries by the supplied ID array and writes CSV
