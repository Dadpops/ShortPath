Polish the UI following v0.4.0: fix support tools, help popout, pin design, accent usage, keyboard shortcuts, entry notes in overlay, and export subfolder UX.

## Decisions made

- **CSS circles for pins**: emoji pins were replaced with 8px CSS circles (`.pin-dot` for entry pins, `.pin-circle` for window pin). Inactive = 1.5px muted outline; active = solid accent fill; hover = 35% accent fill. This replaces all pushpin emoji throughout the app.
- **Accent scoped to folder headers**: accent colors on entry cards (focused state, source badge) were neutralized. Accent is reserved for group header bands, pin state, stars, search match highlights, and action confirmations.
- **Shortcut state in App.tsx**: The keyboard shortcut customization UI in KeyboardPanel directly updates App.tsx's `customShortcuts` state via callback prop. This was the root cause of the original Settings implementation not working — Settings had its own disconnected state.
- **Entry notes inline**: Rather than navigating away from MacroOverlay to see notes, notes are loaded directly in the overlay via `window.shortpath.loadNotes()` and filtered by `entryId`. Deletion updates local state immediately.
