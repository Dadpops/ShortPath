## Decisions made

- `isFirstOnboarding` is a `useRef`, not `useState`, because changing it must not trigger a re-render and it only needs to gate a side effect (opening Help). Using state would have caused unnecessary renders and potential timing bugs.
- `installSeedData` was created separately from `applySeed` because `applySeed` replaces all entries, which is too destructive when a user might already have local entries from before clicking "Set up with Sample Data" in the onboarding.
- "CommandOrControl" is used in `buildGlobalAccelerator` (not "Ctrl") because Electron's `globalShortcut.register` requires the cross-platform accelerator format; in-app shortcuts use "Ctrl" strings.
- Sample data uses `source: "sample"` rather than `source: "local"` so it can be identified and bulk-removed without touching genuine user entries. The CLAUDE.md constraint (sync never touches local entries) is not violated.
